## 디자인 원칙 (1등 노리는 시각화)

### 핵심 컨셉
> **"AI 에이전트 9명이 진짜 뉴스룸처럼 토론하고, 운영자는 모든 걸 손쉽게 통제한다"**

- 에이전트는 **JSON output 출력기**가 아니라 **인격을 가진 동료**처럼 표현
- UI 톤: 차분한 다크모드 + 플러스탭 브랜드 핑크(#ff2e98) accent + 정보 밀도 高
- 운영 옵션은 **풍성하게** (일부는 장식적이어도 OK)
- 마이크로 인터랙션: 호버, 페이드, 슬라이드, 펄스 — 단 과하지 않게

### 톤 레퍼런스
- **Linear** (앱): 다크모드 + 정보 밀도 + 키보드 단축키 노출
- **Vercel Dashboard**: 모노스페이스 + 산세리프 혼용, 색상 절제
- **Discord**: 채팅 버블·아바타 패턴
- **Stripe Workbench**: 사이드바·테이블·검색 — 운영감

### 대화 변환 룰

JSON output 그대로 보여주면 시각적으로 압도되거나 지루함. 다음 룰로 사람 말투로 변환:

| 원본 | 변환 후 |
|---|---|
| `{"confidence_score": 8, "verified_count": 4}` | 팩트체커 ✅: "4개 검증 완료, 신뢰도 8/10. 다음 단계로 ㄱ" |
| `{"decision": "needs_revision", "rejected_critiques": [...]}` | 편집장 📰: "iter 2 토론 종료. 비판 3개 수용, 2개 기각합니다. Writer 재작성." |
| `{"critical_issues": [...5건]}` | 데빌스 😈: "5건 까겠습니다. 1번, 클로징이 진부함..." |

원본 JSON은 **버블 클릭 시 expand**로 접근.

## 페이지별 상세

### 1. `/` 메인 페이지

- 상단: AIDEN 로고 + 타이틀
- 중앙: 카테고리 선택 카드 4종 (맛집·AI트렌드·안전·문화) + "자유 입력" 카드
- 하단: 최근 실행 5건 (썸네일 + 제목 + Judge 평균 점수)
- 우측 사이드바: "지금 실행 중인 세션" 라이브 표시

호버 시 카드 그래픽 패턴 애니메이션. "Generate" 클릭 → `/run/<new_session_id>` 이동, SSE 연결 시작.

### 2. `/run/<session_id>` 트레이스 뷰어

#### 3-column 레이아웃
```
┌─────────────────────────────────────────────────────────────────────┐
│  [헤더] Session ... · 맛집 · running                                 │
├──────────────┬──────────────────────────────────┬────────────────────┤
│ [Stage 진행] │   [채팅 버블 영역]                │  [Now Playing]     │
│              │                                  │                    │
│  Stage 1 ✓   │   🔍 Trend Scout: "..."          │  Writer ✍️         │
│  Stage 2 ▶   │   👥 Audience Analyst: ...        │  iter 2 / 3        │
│  Stage 3 ⏸   │   🎯 Strategy Planner: ...        │  ─────             │
│  Stage 4 ⏸   │                                  │  cost / budget     │
│              │   [Newsroom 토론 영역]            │  elapsed time      │
│  [에이전트   │   ✍️ Writer: "초안 v1..."         │  last latency      │
│   타임라인]  │   ✅ Fact-Checker: "..."          │                    │
│              │   😈 Devil's Advocate: "..."     │                    │
│              │   📰 Editor: "..."               │                    │
└──────────────┴──────────────────────────────────┴────────────────────┘
```

#### 채팅 버블 디자인
- 좌측 아바타 (이모지 + 캐릭터 색상 원형 배경)
- 상단 메타라인: 에이전트명 · 단계 (iter 2) · 타임스탬프 · duration
- 본문: 변환된 사람 말투 메시지
- 하단: `▶ Show raw JSON` / `▶ Highlights` / confidence·score 뱃지
- 도착 애니메이션: 좌측 fade-in + 살짝 bounce
- 다음 에이전트 호출 중일 때 타이핑 인디케이터 표시

#### Stage 2 토론 강조
- iter 1/2/3 사이 구분선 + iter 뱃지
- Writer→FC→DA→Editor 순서를 좌측 timeline 흐름선으로
- DA 메시지: 빨간 좌측 보더 강조
- Editor 메시지: 노란 보더 + 망치 아이콘
- iter 3 강제 종료 시: "⚠️ 강제 종료 (known weakness 노출됨)" 알림 배너

#### 우측 패널 ("Now Playing")
- 현재 실행 중 에이전트 (펄스 애니메이션)
- 진행률 (Stage별 가중)
- 누적 비용 / 예산 (80% 옐로우, 95% 레드)
- elapsed time, 마지막 LLM 응답 latency

#### 좌측 패널 ("Stage 진행")
- 4 Stage 상태 (완료/진행중/대기/실패)
- 클릭 시 해당 채팅 영역 스크롤
- 수직 timeline: 에이전트 호출 순서 + duration bar

### 2-5. ⭐ NEW: Stage 4 진입 시 "Judge 토론 시뮬레이션" 연출

Stage 1~3 완료 후 Stage 4 (Judge Panel) 진입 시점에 **특별한 시각 연출**.

#### 연출 시나리오

기존 채팅 버블 영역 하단에 새로운 섹션 등장. 전환 효과:
- 채팅 영역 상단부가 fade-out + 살짝 위로 스크롤
- 화면 중앙에 굵은 디바이더 등장: `─── 🏆 JUDGE PANEL ──────────────────`
- 디바이더 등장과 함께 BGM cue (선택적, 짧은 ding 사운드 — 토글 가능)
- 우측 패널이 "Judge 진행 상태" 로 교체

#### Judge 토론 채팅 버블 순차 등장

실제로는 동시 호출 (asyncio.gather) 이지만, **UI에서는 의도적으로 순차 등장** (시각적 연출):

```
─── 🏆 JUDGE PANEL ──────────────────

[2초 후]
💎 Gemini 심사위원이 입장했습니다.
💎 Gemini: "평가 시작합니다. final_output.html 분석 중..."
   [⏳ 타이핑 인디케이터 약 3초]
💎 Gemini: "topic_fit 8점, content_quality 7점, ..."
   [차원별 점수 막대가 좌→우로 채워지는 애니메이션]
💎 Gemini: ⭐ 8점 · "트렌드 반영도 양호, 실용성 강함"
   [한 줄 평이 강조 박스로 등장]

[2초 후]
🤖 GPT 심사위원이 입장했습니다.
🤖 GPT: "평가 시작..."
   [위와 동일 패턴]
🤖 GPT: ⭐ 7점 · "interactivity 우수, 톤은 평이"

[2초 후]
🎭 Claude 심사위원이 입장했습니다.
🎭 Claude: "평가 시작..."
   [위와 동일 패턴]
🎭 Claude: ⭐ 6.5점 · "톤이 AI스럽다, 클로징 진부함"

[1초 후]
─── 📊 종합 평가 ───────────────────
[차원별 점수 비교 시각화 영역 등장 — 다음 섹션 참조]
```

#### 연출 디테일
- 각 모델 "입장" 메시지는 회색 시스템 메시지처럼 표시 (Discord "사용자가 채널에 참여했습니다" 패턴)
- 점수 발표는 별 이모지 ⭐ + 큰 폰트로 강조
- 모델별 색상 보더 (Gemini 블루 / GPT 그린 / Claude 오렌지)
- 동시 호출이라 실제 응답 도착 순서는 불규칙. UI에선 강제로 Gemini→GPT→Claude 순서 (브랜딩 일관성)
- 점수 막대 애니메이션: Framer Motion `motion.div` width 0→100% 1초 ease-out
- 한 줄 평은 typewriter 효과 (선택적 — 과하면 토글로 끄게)
- **건너뛰기 버튼**: 우상단 "Skip animation" — 시연 중 시간 없을 때

#### 우측 패널 (Stage 4 진행 중)
```
🏆 Judge Panel 진행

진행률: ▰▰▰▱▱  60%

💎 Gemini  ✓ 8.0 / 10
🤖 GPT     ✓ 7.0 / 10
🎭 Claude  ⏳ 평가 중...

예상 종합: ~7.5
의견 불일치 감지: tone_authenticity
```

평가 진행 중에 실시간으로 점수 채워짐. 마지막 모델 완료되면 종합 화면으로 전환.

#### 3-2. ⭐ NEW: 레이더 차트 (5 차원 × 3 모델)

**가장 임팩트 있는 시각화. 시연 핵심 자료.**

```
┌─────────────────────────────────────────────────────────┐
│ 📊 5차원 × 3모델 평가 분포                                │
│                                                         │
│                topic_fit                                │
│                    10                                   │
│                  /  |  \                                │
│   timeliness    /   |   \   content                     │
│      _trust    /    |    \   quality                    │
│         10 ───┼─────●─────┼─── 10                       │
│                \    |    /                              │
│                 \   |   /                               │
│        tone_auth \  |  /  interactivity                 │
│             10    \ | /    10                           │
│                    \|/                                  │
│                                                         │
│  ─── Gemini  ─── GPT  ─── Claude                        │
│  ═══ 평균 (굵은 선)                                       │
└─────────────────────────────────────────────────────────┘
```

**구현 디테일:**
- 라이브러리: **Recharts `<RadarChart>`** 또는 **D3** (Recharts 우선, 더 단순)
- 5축 (5 차원), 각 모델별로 1개 라인 + 평균선 1개 = 총 4개 라인
- 모델별 색상: Gemini `#4285f4` / GPT `#10a37f` / Claude `#d97757` / 평균 `#ff2e98` (브랜드 핑크, 두꺼운 선)
- 호버 시 해당 꼭짓점에 정확한 점수 툴팁
- 차이가 큰 차원 (outlier 있는 차원) 의 축 라벨에 ⚠️ 아이콘
- 클릭 시 해당 차원의 3 모델 코멘트 비교 모달 등장

**시연 효과:**
- 3 모델이 어느 영역에서 일치하고 어디서 갈리는지 **한 눈에 보임**
- 모델이 만든 5각형 모양 자체가 콘텐츠 강점/약점 패턴
- 발표 자료에 그대로 캡처 사용 가능
