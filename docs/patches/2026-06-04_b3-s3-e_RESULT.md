## §0 자율 결정 로그

- **Monaco Editor 미설치 → textarea + 등폭 폰트(JetBrains Mono) 폴백 사용.** 새 라이브러리 추가 금지 + 배포 영향 회피. 명세 §B2 의 "설치 실패 시 폴백" 경로. **[BLOCKED] 항목으로 §6 에 기록.**
- registry `category` 스키마 = `food | ai_trend | safety | culture | free` (자체 스키마). generate enum (`food | ai-trend | safety | culture | custom`) 과 분리. 이유: registry 는 콘텐츠 분류 메타고 generate input enum 과 lifecycle 이 다름 (registry 는 운영자가 자유롭게 분류, generate 는 UI 카테고리 카드 ID).
- 기존 `.versions/` 폴더를 B3-S3-E 의 `_history` 로 그대로 재사용. (명세는 `_history` 라고 적었지만 의미 동일 — 중복 폴더 생성 회피.) 백업 파일명 패턴 `{stem}_v{n}_{YYYYMMDDTHHMMSS}.md`.
- B5 운영 옵션: 명세는 "백엔드 연동 가능한 것만 실동작" 이라 했지만, **어드민이 서버 전역 default 를 바꿀 endpoint 는 이번 범위 밖**. 모든 wired 항목을 **BLOCKED** 처리 + .env / config 수정 안내 배너 노출. 장식 항목은 토글만 동작.
- registry `_ensure_file()` 가 `data/` 디렉터리를 자동 생성하도록. 명세 §A3 ephemeral 안내는 키 페이지 + RESULT.md 에 명시.
- PromptLoader 캐시 무효화: 기존 코드에 명시적 캐시가 없음 (PromptLoader 인스턴스가 `agent_resources.json` 매핑만 캐시). `concrete_agents.make_agent_callable` 가 system_prompt 를 클로저에 보관하는 경우만 문제 → 다음 run 에서 새 `build_*_agents` 호출 시 새로 읽음. **PUT 후 다음 run 부터 자동 반영. 별도 무효화 hook 불필요.** Scout 만 `dynamic_vars_fn` 경로로 매 호출 파일을 다시 읽음.

## ⚠ 배포 환경 ephemeral 안내 (RESULT.md 필수 명시)

배포 환경에서 admin 저장 기능은 **ephemeral** 입니다. 시연은 로컬 권장:

- `backend/agents/prompts/_defaults/` 와 `.versions/` 는 **컨테이너 파일시스템**에 저장됩니다. Railway 재배포 시 소실.
- `data/topic_registry.json` 도 컨테이너 파일. 재배포 시 빈 목록으로 초기화.
- `RuntimeKeyStore` 는 프로세스 메모리. 재시작 시 소실 → .env 키로 복귀.
- v2 에서 Volume / DB / 외부 시크릿 매니저로 이관 예정.
